HTTP权威指南笔记

发表于 2018-02-11 • 分类于 网络世界

学习HTTP，深入了解URL网络世界。

第一章 HTTP概述

HTTP

web客户端和服务器

资源

媒体类型MINE

URI统一资源标示符

URL统一资源定位符

URN统一资源名

一个HTTP事务由一条请求命令和一个响应结果组成，通过名为 HTTP报文的格式化数据块进行的

HTTP方法：GET PUT DELETE POST HEAD

状态码：200OK 302Redirect 404Not Found

web页面是一组资源的集合，包含很多请求。

报文：起始行+首部字段+主体

连接：TCP/IP是全世界的计算机和网络设备常用的层次化分组交换网络协议集。隐藏了各种网路和硬件的特点及弱点，使各种类型的计算机和网络都能进行可靠的通信。

=>物理层（物理网络硬件）

=>数据链路层（网络特有的链路接口）

=>网络层（IP）

=>传输层(TCP)

=>应用层（HTTP）

  1. 浏览器从URL中国解析服务器主机名；
  2. 浏览器的主机名转换成服务器的IP地址；
  3. 浏览器解析端口号
  4. 浏览器建立与web服务器的TCP连接；
  5. 浏览器向服务器发生HTTP请求；
  6. 服务器向浏览器回送HTTP响应报文；
  7. 关闭连接。浏览器显示文档。

HTTP1.0得到广泛使用，HTTP1.0+加入了很多非正式的HTTP扩展，HTTP1.1校正了HTTP设计中的结构性缺陷，明确语义，引入了重要的性能优化措施，并删除一些不好的特性。HTTP2.0大幅优化性能和提供更强大的服务逻辑远程执行框架。

web的结构组件：

代理：存在客户端和服务器间的HTTP中间实体。

缓存：HTTP的仓库。

网关：连接其他应用程序的特殊web服务器。

隧道：对HTTP通信报文进行盲转发的特殊代理。

Agent代理：发起自动HTTP请求的半智能Web客户端。

第二章 URL与资源

URL=方案+域名+资源路径+其他组件

相对URL: ./hammers.html是不完整的。

基础URL+相对URL=绝对URL。

找寻基础URL的可能有：

在资源中显性提供；

封装组员的基础URL；

损坏了的URL。

解析相对引用并引用资源的算法

自动扩展URL：主机名扩展；历史扩展

URL是可移植的；还得是完整的。因此要设计URL显然是意见麻烦的事情。

通过转义序列，就可以用US-ASCII字符集的有限子集对任意字符值或数据进行编码。
