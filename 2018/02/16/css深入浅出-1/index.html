      堆叠上下文

      一、堆叠顺序

  1. 背景和边框
    通过设置背景色的rgba来查看，background在border后面，border更靠近用户
  2. 文字（内联元素）与边框
    通过加上text-indent: -20px;还可以看见文字/内联表明文字在border上面
  3. parent的文字与div.child
    设置child的margin-top: -10px;看见文字（内联）盖住了child，所以内联元素离用户更近
  4. parent的文字与child的文字
    把文字改为实心的图案加上margin-top可以看见child的文字盖住了parent的文字
  5. 浮动元素
    parent的文字比浮动元素更靠近用户（text-indent: -4px;）
    浮动元素比块级元素更靠近用户
    浮动元素的文字比不过parent的文字
  6. position
    position: relative可以盖住浮动元素和parent的文字，是除了absolute的最高级别
  7. z-index只对positio不是static的元素生效，默认是auto。
  8. 相同级别的relative后面的盖住前面的，可以通过改变z-index可以改变堆叠顺序。比z-index: 0大的高。
  9. 没有设置z—index时absolute比relative高，相同absolute堆叠顺序也可以z-index改变，absolute和relative也可以。
  10. 负z-index出现在background后面，不过也是要设置position

二、堆叠上下文（The stacking context）

  1. parent的z-index不为auto时，parent的子元素中的z-index为负的元素都会发生堆叠上下文，跑到div的后面，border的前面。

  2. 文档没有概念，只说特征和例子，和bfc类似

  3. 堆叠上下文产生条件

    • 文档的根元素（HTML）。
    • 具有position“绝对”或“相对” z-index值和“自动”以外的值的元素。
    • position值为“固定”或“粘滞”的元素（适用于所有移动浏览器，但不适用于旧桌面）。
    • 元素是flex（flexbox）容器的子元素，其z-index值不是“auto”。
    • opacity值小于1的元素（请参阅不透明度规范）。
    • 元素的mix-blend-mode值不是“正常”。
    • 具有以下任何非“无”值的元素：
    • transform
    • filter
    • perspective
    • clip-path
    • mask/ mask-image/mask-border
    • 具有isolation值“隔离”的元素。
    • 具有-webkit-overflow-scrolling“touch”值的元素。
    • 带有will-change值的元素，指定将在非初始值上创建堆栈上下文的任何属性

  4. 基础的堆叠上下文是HTML，设置了position的普通情况下z-inde值大的越靠近用户，但是如果父元素的z-index大于另外一个z-index的值，那么该父元素的子元素都比另一个父元素的子元素更靠近用户。
